# Lab 2.6: Geometric Perceptron

## Work Description
The goal of this laboratory work is to implement a simple artificial neural network (Perceptron) to solve a classification problem. Specifically, the model is designed to determine whether a point $(x, y)$ in a unit square lies within a specific nested square defined by the boundaries $[0.25, 0.75] \times [0.25, 0.75]$.

Unlike training via backpropagation, this lab focuses on understanding the **architecture and logic** of a neural network by manually defining weights that correspond to geometric constraints.

## Mathematical Model
The problem is solved using a two-layer architecture:

1.  **Input Layer**: Receives coordinates $(x, y)$.
2.  **Hidden Layer (4 neurons)**: Each neuron represents one linear inequality (a side of the square):
    *   $n_1: x \ge 0.25$
    *   $n_2: -x \ge -0.75$ (equivalent to $x \le 0.75$)
    *   $n_3: y \ge 0.25$
    *   $n_4: -y \ge -0.75$ (equivalent to $y \le 0.75$)
    *   **Activation**: Heaviside step function $f(z) = 1$ if $z \ge 0$, else $0$.
3.  **Output Layer (1 neuron)**: Performs a logical **AND** operation. It outputs $1$ only if all four hidden neurons are active ($1$). This is achieved by setting weights to $[1, 1, 1, 1]$ and a bias to $-4$.

## Implementation & Results
- **Architecture**: A hard-coded network that mimics a decision tree of linear boundaries.
- **Visualization**: The resulting plot shows a clear red square (class 1) inside a blue field (class 0), perfectly matching the theoretical square.
- **Accuracy**: As the weights are precise geometric translations, the accuracy is 100% relative to the intended ground truth.
